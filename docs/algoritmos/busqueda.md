# Búsqueda

Buscar es responder "¿está este elemento y dónde?". El costo depende muchísimo de si los datos están ordenados y de qué estructura los respalda: desde recorrer todo de principio a fin (lineal) hasta saltar directo con una tabla hash. Aquí están las cuatro estrategias principales con sus ejercicios y una comparación para elegir.

> [!NOTE]
> La búsqueda binaria y la interpolación exigen datos **ordenados**. Si no lo están, ordena primero (ver [Ordenamiento](ordenamiento.md)) o la respuesta no tendrá sentido.
